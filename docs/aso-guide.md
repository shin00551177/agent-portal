# ASO 分析 ガイド

## 概要

Agent Portal の ASO 機能は、Apptweak のキーワードデータと App Store Connect / Google Play Developer API を組み合わせて、アプリのストア最適化（ASO）を自動化・半自動化するシステムです。

**対応アプリ**：Twomi / AI AVATAR / SOULRiZA / KING Together / Education / パチナビ

---

## 1. ダッシュボード

`/aso/{アプリID}` でアクセス。

### 集計期間

データが取得された期間を表示します。前回同期との比較期間も自動で計算されます。

### アプリ指標

| 指標 | 説明 |
|---|---|
| DL数 | 本日のダウンロード数（Apptweak 推計） |
| 売上 | 本日の売上（USD、Apptweak 推計） |
| 評価 | 平均評価スコアと総件数 |
| App Power | Apptweak 独自の総合スコア（0〜10）。低いほど検索露出が弱い |

### キーワード順位テーブル

| 列 | 説明 |
|---|---|
| キーワード | 監視対象のキーワード |
| 順位 | 現在の検索順位（圏外 = 501位以下） |
| 変動 | 前回同期からの順位変化（↑上昇 / ↓下降 / →変化なし） |
| 検索ボリューム | そのキーワードの月間検索数（0〜100スケール） |
| 競合密度 | そのキーワードで現在ランクインしているアプリの強さの総量（0〜100）。低いほど参入しやすい |
| 推奨戦略 | キーワードの状況に応じた推奨アクション（下記参照） |

**推奨戦略タグの意味**

| タグ | 条件 | 意味 |
|---|---|---|
| ✓ 守る | 10位以内 | 現在の順位をキープする |
| ↑ 伸ばす | 11〜50位 | 施策を続けてさらに上位を目指す |
| 🎯 ねらう | 圏外 × ボリューム高 × 競合密度低 | 今すぐ攻めるべきキーワード |
| 激戦区 | 競合密度 70超 | 強豪が占拠しており参入難度が高い |
| 様子見 | その他 | 優先度を下げて経過観察 |

## 2. データ更新

ダッシュボード右上の「データ更新」ボタンで Apptweak からデータを取得します。

### 通常更新

「データ更新」ボタンを押すと、今日のスナップショットデータを取得します。取得後は **自動で AI 分析も実行**されます。

### 期間指定取得

「期間指定 ›」を押すと日付ピッカーが表示され、任意の期間の順位推移を取得できます。

- 開始日〜終了日を指定して「この期間を取得」
- 週次サンプリングでトレンドテーブルを表示
- **分析は実行されない**（スナップショットのみ）

### 自動実行

毎週月曜 10:00 JST に自動でデータ更新 + AI 分析が実行されます（launchd 設定済み）。

## 3. AI 分析・改善提案

### 分析の実行

「Claude で再分析」ボタンを押すと、最新のデータをもとに Claude が分析を実行します。データ更新後は自動で実行されます。

### 提案の構成

各提案は必ず以下の3点セットで構成されます：

| セクション | 内容 |
|---|---|
| **結果** | 現状の数値・事実のみ（判断を含まない） |
| **原因分析** | なぜその結果になっているかの推論 |
| **ネクストアクション** | 具体的な変更内容と期待される効果 |

### 承認・差し戻し

- **🚀 よし、やろう！** → 承認。App Store Connect / Google Play へ自動で変更を申請します
- **🤔 ちょっと違う…** → 差し戻し。理由を入力して却下できます（入力は省略可）

### App Store Connect への自動反映

承認後、キーワード・サブタイトル・説明文の変更が自動で申請されます。

**⏳ バージョン待ち**（iOS）  
App Store Connect に編集可能なバージョン（審査準備中）がない場合、提案は「バージョン待ち」状態で保留されます。VN チームが新バージョンを作成したあと「🔄 再試行」ボタンで適用できます。

## 4. AIに聞く

データについて自由に質問できるチャットボックスです。

**クイック質問の例：**
- 「App Power がこんなに低いのはなぜ？」
- 「今すぐできる改善アクションは？」
- 「どのキーワードを優先的に攻めるべき？」

テキスト入力で独自の質問も可能です。Enter キーまたは「聞く」ボタンで送信。

---

## 5. キーワード管理

ダッシュボードの「キーワード管理」セクションでキーワードを追加・削除できます。

- **優先度 高**：より頻繁に分析対象となります
- **優先度 中**：通常の分析対象

Enter キーまたは「追加」ボタンで登録。× ボタンで削除。

## 6. 画像・バナー差し替え（β）

⚙ 設定ページ内の「画像・バナー差し替え」セクションから操作します。

詳細は [aso-image-upload.md](./aso-image-upload.md) を参照してください。

**⚠️ ドライランについて**  
デフォルトは **ドライラン ON**（テストモード）です。実際に審査提出する場合は OFF に切り替えてから実行してください。

## 7. 設定（⚙ アイコン）

各アプリの設定ページ（`/aso/{アプリID}/settings`）で以下を管理できます。

### エージェント設定

ワークフローの ON/OFF を切り替えます。OFF にすると自動 sync が停止します。

### ガバナンス設定（AI AGENT 行動規範 v0.1 準拠）

| 設定項目 | 説明 |
|---|---|
| エスカレーション閾値 #8 | この条件を超えたとき Agent が自律判断せず通知 |
| 停止条件 #12 | 連続エラー数がこの値を超えたとき Agent が自動停止 |
| オーナー不在時の動作 #13 | 一時停止 / 保守的実行 / 継続実行 から選択 |

### Agent 台帳（運用ポリシー §5）

対象業務・運用責任者・承認日・承認者・月間削減工数を記録します。全体設定の `/settings` ページで一覧確認できます。

## 9. クレジット使用量の目安（Apptweak API Small）

月間 100,000 クレジット（トライアル）。週次自動実行の消費量の目安：

| 操作 | 消費量（目安） |
|---|---|
| キーワード順位取得（10KW） | 〜180 cr |
| アプリ指標取得 | 〜1,021 cr |
| キーワードボリューム・競合密度 | 〜200 cr |
| **週次合計（1アプリ）** | **〜1,400 cr** |
| **月次合計（6アプリ）** | **〜33,600 cr** |

月間 100,000 cr に対して約 34% の消費。余裕あり。
